# 1. ビジネスアイデアの概要

## 農業収益化エンジン **APE (Agri-Profit Engine)**

> 「何を作れば儲かるか」をAIが教えてくれる
> 新規就農者向け経営支援プラットフォーム

### 3つの科学的エンジン

- 🔬 **栽培物理エンジン**: 土壌×気象データで「育つもの」を診断
- 📊 **経営経済エンジン**: 市場価格×コストで「儲かるもの」を計算
- 🧠 **行動変容エンジン**: ナッジ理論で「実際に動ける」ように後押し

---

# 2. 逆説・課題

## 農業の「3つの博打」問題

| 不確実性                   | 現状             | 結果               |
| :------------------------- | :--------------- | :----------------- |
| **何を作る？**       | 経験と勘で決める | 失敗のリスク大     |
| **いくらで売れる？** | 市場任せ         | 豊作貧乏           |
| **どこに売る？**     | JA一択           | 安値で買い叩かれる |

### 🔥 逆説

> 「情報がないから失敗する」のに、
> **既存サービスは"作った後"の管理ツールばかり**
> → 「作る前」の意思決定を支援するプレイヤーがいない

# 4. このビジネスへの想い

## 農家として、市場で働いて、見えたこと

> トマト農家に生まれ、椎茸・イチゴ・ネギ…と挑戦し、**何度も失敗**した。
> 青果市場に勤め、**「気候が変わっても設備投資のせいで作物を変えられない」農家**を見た。
> 実体験として、農協の職員さんに「他に作物を作るとしたら何が良いですか？」と聞いたことがあるけど「危険なのでおすすめできませんと言われた」
> 次にJAの様な組織の青果卸売会社で働いてるときにも同じ質問をしたけど、ほんとにそれでうまくいく？今まで通りの作物を作るのが一番と流されて終わったこともある。
> さらに市場で経験豊富な上司にも同じように「儲かる作物ってなに？」と聞いたところ、分からないと言われた。
> 農協の職員もへたなアドバイスをして責任はとれないし、市場で経験豊富な人でも野菜の相場を正確に読むのは難しいから分からないって言ってるしでみんな答えが分からない問題なんだなと思った。

### だから作りたい

**「過去のしがらみではなく、未来のデータで経営を決められる」世界**

# 補足: AIの活用方法

## このアイデア作成でAIをどう使ったか

1. **文献調査**: 農業経済学・行動経済学の論文20本以上をAIで要約・整理
2. **計算ロジック設計**: Yield Gap理論やSERFモデルの数式をAIと壁打ちして定義
3. **農研機構データ取得**: PDFスクレイピングスクリプトをAIが自動生成
4. **このスライド**: Marp形式でAIが即時生成

> AIは「相棒」として、調査・設計・制作のすべてに活用しました
